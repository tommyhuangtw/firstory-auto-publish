# Threads 高流量寫作研究 — 爆文 Playbook + 工具取經

> 研究日期：2026-06-26 ｜ 來源：① 12,157 篇公開 Threads 貼文資料集（`public_posts_raw_rows_threadify.csv`，165 帳號）② AK-Threads-Booster GitHub skill ③ Threads 演算法外部資料

---

## 0. 一句話總結

**Threads 爆文 = 狠 hook 開頭（刪掉反差／數字結果／脆弱自白）＋ 一句一行的手機節奏 ＋ 乾貨後置到留言區 ＋ 真實第一人稱數字 ＋ 互惠型結尾。**
利基（AI/接案）最強公式：「我用 AI 達成具體數字 → 5 個指令在留言👇 → 收藏起來」。

---

## 1. 資料面發現（量化）

- 互動極度長尾：p50=15、p90=173、p99=2360、max=96,579（本資料以 ♥讚 為主）。
- **高互動文略短**：前 10% 中位數 103 字 vs 其餘 115 字。
- **有媒體相關性**：前 10% 有圖/影片比例 9% vs 其餘 2%（高 4.5 倍）——但**榜上多數仍是純文字**，金句純文字能爆到 9.6 萬。媒體加分但非必要。
- 開頭問句在最高互動文反而較少（0.09 vs 0.15）——「問句開場」不是萬靈丹。

## 2. 開頭 Hook 六大類型（引用真實貼文）

| 類型 | 例子 | 為何抓人 |
|------|------|----------|
| **刪掉/丟掉反差**（最會爆） | 「我把 Duolingo 刪掉了。因為 ChatGPT 教我韓文…直接輾壓我前面苦撐的兩年。」 | 放棄大家都在用的東西 → 認知衝突 |
| **數字/結果前置** | 「我用 ChatGPT…35000 機票砍到 20000。」「投了 60 多份履歷。沒人回。」 | 具體數字=可信+好奇缺口 |
| **脆弱/自嘲自白** | 「23 到 27 歲是我人生最焦慮的階段…薪水只有 28k」 | 先示弱→共鳴→翻身才有重量 |
| **好奇缺口/賣關子** | 「不小心開啟了『神模式』」「被很多人忽略的功能」 | 標籤化未知製造資訊缺口 |
| **衝突/立場宣告** | 「我只服 PAPAYA 老師，不服來辯！」 | 點名/站隊 → 留言區辯論 |
| **場景帶入（對話體）** | 「今天聽到 A 同事問新人：你是台大純血喔？」 | 像偷聽八卦，自然往下滑 |

## 3. 形式/結構

- 四種主結構：**單句暴擊（金句）／短故事弧／清單懶人包／編年史 Before-After**。
- **一句一行 + 大量空行**（手機呼吸感）；正文常 ≤120 字。
- **乾貨後置到留言區**是最強格式技巧：「指令放留言區👇」「整理在留言，這篇收藏起來」→ 衝留言數+停留 → 演算法加權。
- Emoji 只當功能用（👇指留言、✅列結果、📍清單錨點；情緒字一篇 2-3 個）。

## 4. 結尾/CTA

- 衝**回覆** → 開放提問（「你會怎麼做？」）。
- 衝**留言/收藏** → 「乾貨在留言 + 收藏起來」。
- 衝**讚/分享** → 金句收束（常完全無 CTA）。
- CTA 用「互惠」（留言我就送你懶人包）而非「乞求」（拜託按讚追蹤）。

## 5. 題材（利基 AI/接案最會爆的框法）

1. **工具反差**：「我把 X 刪掉了 / 我已很久沒用 ChatGPT，都用 Gemini」——把工具偏好寫成個人決定。
2. **個人結果數字化**：「我用 X 達成具體數字」，不是「X 很強」。
3. **工具即故事**：不講功能，演場景（拖曳 Excel 朋友愣住）。
4. **唱反調**：「AI 不會取代你，用 AI 的人會」「N 個指令不如一個好問題」——反主流本身是爆點。
5. **N 個指令/免費資源懶人包**（衝收藏，但純連結會被貼「樣板文/騙讚」標籤，要包個人故事）。
6. **接案/職涯編年史**：年齡軸逐行推進。

## 6. 信任感

- **具體數字 > 形容詞**（28k、12GB、進步 300 分）。
- **承認失敗/示弱在前**。
- **第一人稱實測語氣**（「我自己測過最有感」）。
- **誠實的不確定**（「第二個我不知道中文叫啥」）。
- **點真名**（真實工具版本/品牌）。
- ⚠️ 純連結轟炸無故事 = 低信任，會被公開點名「騙讚」。

## 7. 給 AI 寫手的 18 條可執行規則

1. 第一行 ≤25 字就要有鉤子（反差宣告或數字前置）；禁「今天想跟大家分享」。
2. 第一人稱 + 至少一個硬數字；「很有效」→量化結果。
3. 工具當配角，框架是「我達成了什麼」；能演成場景就演。
4. 乾貨後置到留言；正文 ≤120 字。
5. 一句一行、大量空行；禁三行以上長段。
6. 結尾二選一（開放提問 ＸＯＲ 收藏+留言指令），不要兩個都塞。
7. 示弱在前、翻身在後。
8. 可選爆點：唱反調（附真實理由，不為反而反）。
9. Emoji 只當功能用，一篇情緒字 ≤3。
10. 可純文字，不為發圖硬配圖（只有結果截圖/前後對照值得配）。
11. 點真名增加可信，不用「某個 AI 工具」。
12. 保留一句小坦白，避免全知業配感。
13. 避免純連結清單當主力（要包個人故事/判斷觀點）。
14. 給方法取吸睛名（「神模式」「隱藏用法」）。
15. 成長/職涯用編年史結構。
16. 每篇鎖定一個情緒（好奇/共鳴/不甘/爽/憤怒）。
17. CTA 用互惠而非乞求。
18. 長度對應目標：衝量=金句 20-50 字；衝信任=故事弧/編年史。

## 8. Threads 演算法（外部驗證）

- **回覆(replies)是最重的單一訊號**；有意義的留言權重 > 簡單回應 → 印證「乾貨後置留言 + 開放提問」。
- 停留時間、互動速度（首小時回覆）也是訊號。
- **真實/原創內容 > 套路/搬運/growth hack**（2025-26 強化）→ 印證「不抄舊文、真實第一人稱」。
- 推薦式 feed，小帳號也能觸及大量陌生人（陌生人契合度 stranger-fit 很重要）。
- 圖片據稱比純文字多 60% 觸及（與本資料「純文字也能爆」並存：媒體加分非必要）。

## 9. AK-Threads-Booster 取經點（GitHub skill）

它是一套「數據驅動」的 Threads 內容系統(10 個指令)。我們已有 /voice、/draft；**值得借鏡的**：

| 它的機制 | 我們可以借的 |
|----------|--------------|
| **Freshness Gate**（檢查角度是否已過度飽和） | 在寫手加「新鮮度檢查」——對照已寫過的、避免重複(正中 Tommy 的痛點) |
| **/topics 主題引擎**（依歷史成效+留言問題+語意分群避免重複+飽和度，推薦「值得發」的題） | 把骰靈感升級成「智慧推薦題目」 |
| **Anti-Voice Forbidden Zones**（明確禁區） | 我們已做(去金句)，可擴成一份「禁止清單」 |
| **Cognitive layer**（核心信念/判斷框架/觀點張力） | 把「mindset」做成獨立資產，讓延伸更有靈魂 |
| **/analyze 發布前決策層**（演算法合規、陌生人契合、AI 偵測風險、瓶頸診斷） | 草稿生成後加「爆文體質評分 + 改進建議」 |
| **/predict**（用相似歷史貼文估 24h 互動範圍） | 用 CSV/自己歷史估「這篇大概多少互動」 |
| **/review 回饋迴圈**（實際 vs 預測，越用越準） | 發布後回填互動 → 持續校準 |

## 10. 建議：怎麼接進 Tommy 的寫手

**v1（最有感、低成本）**：把第 7 節 18 條規則做成一份「**爆文 Playbook**」資產，寫手加一個「**爆文模式**」開關——開啟時把 playbook 規則注入 prompt(hook 公式、一句一行、乾貨後置、互惠 CTA、鎖一個情緒)。

**v2（進階）**：
- **草稿後評分**(/analyze 概念)：生成後給「爆文體質分數 + 為什麼 + 一鍵套用建議」。
- **Freshness Gate**：對照已生成/已發草稿,警示「這個角度你最近寫過」。
- **預測互動**：用這份 CSV(165 帳號高互動)當基準,估體質。

**v3（生態）**：發布後回填真實互動 → 學習迴圈,讓推薦與評分越用越準。
