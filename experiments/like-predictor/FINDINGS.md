# 社群流量 Predictor — 實驗結論

**日期**: 2026-06-26 · **Branch**: `feat/like-predictor`

## TL;DR

**贏家 = Approach A（古典 ML：TF-IDF + GBDT）**。在你的兩個實際用途——「比較草稿挑最好」和「判斷會不會爆」——上都最準，而且零 API 成本、最快、可解釋。

## 三方法評測結果

> 為公平比較，C(LLM) 只在 200 筆抽樣上評測，所以 A/B 也在**同一份 200 筆**上重測（下表 `(sample)` 列才是三方對齊的蘋果比蘋果）。

| 方法 | Spearman(整體) | **Spearman(作者內)** | Pairwise Acc | ROC-AUC | PR-AUC |
|------|:---:|:---:|:---:|:---:|:---:|
| **A 古典 (sample)** | 0.186 | **0.354** 🏆 | 0.626 | **0.723** 🏆 | **0.278** 🏆 |
| B Embedding (sample) | 0.166 | 0.318 | **0.643** | 0.668 | 0.205 |
| C LLM (sample) | 0.252 | 0.151 | 0.381 ⚠️ | 0.706 | 0.132 |

（A/B 在完整 2390 筆測試集上：A 作者內 ρ=0.291、B ρ=0.306，趨勢一致。）

## 怎麼解讀

**「比較草稿挑最好」= 作者內 Spearman + Pairwise Acc**
- A 古典 **0.354** 最強，pairwise 0.63（每 10 次挑對約 6.3 次）。
- ⚠️ **LLM(C) 在這項輸到脫褲**：pairwise 只有 **0.38，比丟硬幣還差**。原因：LLM 判的是「文章絕對好壞」，但同一個帳號的草稿之間差異很細膩，LLM 抓不到帳號專屬的眉角；它只會說「這篇寫得通順」≠「這篇對你的受眾會爆」。

**「會不會爆」= ROC-AUC / PR-AUC**
- A 古典 ROC-AUC **0.723**、PR-AUC **0.278**（base rate 僅 8%，0.278 算有料）最佳。
- LLM ROC-AUC 0.706 不差（有世界知識，能分辨「普遍好/壞」的內容），但 PR-AUC 低、precision 差。

## 為什麼古典 ML 贏

1. **char n-gram (char_wb 2-4)** 對中文 Threads 抓「用詞/句式/語氣」很有效，剛好是同帳號草稿間的差異所在。
2. GBDT 能吃結構特徵（開頭鉤子長度、問句、emoji、發文時段、有無媒體）。
3. 目標用「作者相對基準」設計，移除粉絲數干擾 → 模型專心學內容。

## 限制（誠實揭露）

- 訊號中等強度（作者內 ρ≈0.35），代表「方向對、但別當神諭」。它能在 3-5 個草稿裡幫你**排序**、把明顯弱的篩掉，但**無法給你準確讚數**（資料缺粉絲數，本來就做不到）。
- 「爆」定義為作者自己 P90；冷啟動（沒在訓練資料出現過的新帳號）不在範圍內。
- 資料以他人公開貼文為主（12k 篇 / 158 帳號）；你自己的 521 篇可作為下一步的個人化微調來源。

## 建議：用 A 當輔助工具

把 Approach A 包成 `predict_draft(text, author)` → 回傳 (相對分數, 爆文機率)。
用法：你寫 3-5 個草稿 → 工具排序 + 標出最可能爆的那篇，當**第二意見**，不是取代你的判斷。

下一步（若要產品化）：
1. 用你自己的 521 篇做 personal fine-tune / 重新計算 author baseline。
2. 接進 `/inspiration` 或 voice-writer，草稿生成後即時打分排序。
